System

系统设置

配置站点、登录方式、Agent 接入和通知行为。

站点

展示名称和前台访问控制。

管理员登录

配置后台允许使用的登录方式和管理员身份范围。

OAuth2 管理员 GitHub / Gitee 登录后，只允许名单内账号进入后台。 {{if ts .Conf.Oauth2.Admin}} 已配置 {{else}} 未配置 {{end}}

填写 OAuth2 登录名，多个账号用英文逗号分隔。

本地备用账号 无需第三方 OAuth，可用于首次部署或外部登录不可用时进入后台。 {{if .Conf.Auth.Local.Enabled}} 已启用 {{else}} 未启用 {{end}}

首次启用本地登录时必须设置密码。

建议至少保留一种可用登录方式。OAuth2 的 Client ID 与 Client Secret 仍在 data/config.yaml 中维护。

Agent 接入

控制后台复制出的 Agent 安装命令所使用的服务地址和连接协议。

Agent 连接入口 生成安装命令时会把这个地址写入 Agent 参数，用于连接 dashboard/gRPC 单端口服务。
{{if ts .Conf.Agent.InstallHost}} 自定义地址 {{else}} 跟随访问地址 {{end}}

留空时，复制安装命令会自动使用当前 dashboard 的访问域名和端口。

通过 HTTPS 反代或 443 端口接入时建议开启；直连 HTTP/h2c 时保持关闭。

节点 IP 变更通知

当 Agent 上报的节点 IP 与上次记录不一致时，向管理员发送提醒。

节点 IP 变化提醒 适合发现公网出口、NAT、云主机弹性 IP 或网络线路发生变化。
{{if .Conf.EnableIPChangeNotification}} 已启用 {{else}} 未启用 {{end}}

通知会通过已配置的通知方式发送；未配置通知方式时不会产生外部消息。
