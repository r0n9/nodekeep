服务监控说明

执行方式

调度dashboard 每 3 分钟分发一次监控任务 执行任务会发送给在线 Agent 轮询执行 结果Agent 回传成功状态、延迟和错误信息 状态页结果会写入历史记录，用于计算可用率

监控类型

HTTP-GET目标填写完整 URL，例如 https://example.com
HTTPS会同时检查 SSL 证书到期、变更和证书错误
ICMP-Ping目标填写主机名或 IP，不带端口
TCP-Ping目标填写主机名/IP + 端口，例如 example.com:443

通知行为

SSL证书错误、7 天内过期、证书变更会直接发送通知 规则SSL 通知不需要配置告警规则 失败HTTP、ICMP、TCP 普通失败当前只记录到服务状态历史

服务监控依赖在线 Agent 执行；如果没有在线 Agent，监控任务不会被执行。
