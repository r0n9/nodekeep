计划的格式为：* * * * * 分 时 天 月 星期，详情见 计划表达式格式
命令：就像写 shell/bat 脚本一样，但是不推荐换行，多个命令使用 &&/& 连接，如果遇到 xxx 命令找不到，可能是 PATH 环境变量的问题，Linux 主机在命令开头加入 source ~/.bashrc或者使用绝对路径执行。

计划任务说明

执行链路

调度dashboard 按计划表达式触发任务 下发任务会发送给所选在线 Agent 执行Agent 在本机执行命令并回传结果 记录dashboard 更新最后执行时间和最后结果

配置字段

Scheduler标准 5 段 cron：分 时 日 月 星期
Command下发到 Agent 的本机命令
Servers通过 ID 或名称选择执行服务器
PushSuccessful开启后成功执行也发送通知

通知行为

成功仅在开启“推送成功的消息”时通知 失败命令执行失败会通知并带回执行日志 离线目标 Agent 离线会通知无法执行 发送失败任务下发失败会通知错误原因

命令环境

Linux/macOS通过 sh -c 执行 Windows通过 cmd /c 执行 超时单次命令最长执行 2 小时 PATH命令找不到时使用绝对路径或加载环境变量

示例

{
  "Scheduler": "0 3 * * *",
  "Command": "source ~/.bashrc && /usr/local/bin/backup.sh"
}

计划任务依赖目标 Agent 在线；多个命令建议使用 && 或 & 连接，并优先使用绝对路径。
